EL表达式和jstl全

JavaEE    Coselding    2014-11-17 01:08

本文为博主原创，允许转载，但请声明原文地址：http://www.coselding.cn/article/2014-11-17/-186085139.html

1.$ {...}中间填入域中的对象名，可用"."实现对对象中的对象的访问，支持对象名，map键值对，request中参数等方式进行访问；$ {person.address.city}
2.其内部会对各个域进行查找，找到符合条件的进行输出，找不到就输出""空字符串，对页面没有影响；
3.EL表达式取集合中数据：$ {list[1].name}这种方式只能取集合中的特定数据，需要迭代输出的要用jstl
4.用el表达式在取数据时，通常用"."，"."取不出来时，用"[]"
5.$ {pageContext.request.contextPath }获取当前web应用路径或是web名称
6.EL表达式支持基本的算术运算和关系运算，=，!=，>，<，>=，<=，and(&&)，or(||)，not(!)，
7.Jstl要导入jstl.jar和standerd.jar两个开发包，JSP页面中要<%@ taglib uri="" prefix="" %>导入标签库,uri填standerd包中META-INF中的c.tld说明的标签库所绑定的URI地址，prifix填"c"
8.<c: foreach var="" items="" >  <c:foreach var="person" items="$ {list}"> 
     var相当于增强for循环中的迭代子数据，items是需要迭代的集合
注意：若迭代的是map集合，则迭代得到的子数据是map.entry，entry.key得到map键， entry.value得到map值
9.<c:if test="" >   <c:if test="$ {date!=null }">相当于条件表达式，符合条件才会输出标签中的内容
10.Standard.jar常用标签库位置：
   C.tld：核心标签库，常用的标签库所在位置
   Fmt.tld：国际化标签库
   Sql.tld：数据库标签库
   X.tld:XML标签库
   Fn.tld：EL函数库
11.常用标签;
  A.<c:out></c:out>：输出字符串内容，在out对象中输出
      value:要输出的内容
      Escape:xml标签转义输出
      Default:value没有值时的默认输出内容
  B.<c:set></c:set>：给JavaBean对象或者map设置值
  两种搭配：
    Value：要设置的值
    Var：要设置的属性名
    Scope：该属性所在域
  和
    Value：要设置的值
    Target：要设置值的Bean对象或map
    Property：要设置值的Bean对象的属性或map
  C.<c:remove></c:remove>删除域中的属性值
    Var:要删除的属性名
    Scope:要删除属性的域
  D.<c:catch></c:catch>捕获嵌套在标签体中的异常
    Var：异常对象，保存在page域中，异常对象中有message，cause，stacktrace三个属性
  E.<c:if></c:if>条件判断
    Var:测试结果的属性名
    Scope:测试结果存在的域
    Test:条件测试表达式
  F.  实现if-else条件选择
   <c:choose>
     <c:when test="$ {user==null }">
        user is null
     </c:when>
     <c:otherwise>
        user is not null
     </c:otherwise>
   </c:choose>
  G.<c:forEach></c:forEach>：循环迭代
    var：迭代的元素
    items：迭代的容器
    begin：迭代开始参数
    end：迭代结束参数
    step：迭代步长
  注：得到list长度要用$ {fn:length(list)}，因为他的size()不能通过反射得到，不能直接用EL表达式。
  H.<c:url></c:url>：URL地址重写，对URL附加参数等操作
    Value：要构造的URL，value中的"/"就表示webapp的根目录，只需
       要在/后面添加/后面的路径内容
    Var：url对象名
    Scope：url对象存在域
    没指定scope和var的话，就是以字符串形式输出到页面。
    Url后面会自动添加上服务器记录的session id
    往url后面添加请求参数：
    <c:url value="/servlet/ServletDemo" var="servlet">
      <c:param name="name" value="中国"></c:param>
    </c:url>
  I.<c:redirect></c:redirect>：重定向
    Url：重定向目标url
    Context：重定向到同一web服务器下其他webapp时，指定其他
         webapp名称
J.
